# AssetFlow: Enterprise Asset & Resource Management System

AssetFlow is a centralized ERP platform designed to simplify the tracking, allocation, and maintenance of physical assets and shared resources. By digitizing manual workflows, AssetFlow provides organizations with real-time visibility, role-based governance, and structured lifecycle management.

## 🚀 Vision & Mission
Our mission is to build a user-centric, responsive application that streamlines asset management. We aim to replace inefficient spreadsheets with a robust system that handles asset lifecycles, resource bookings, and maintenance workflows without unnecessary complexity.

## 🔑 Key Features
*   **Role-Based Access Control:** Secure management for Admins, Asset Managers, Department Heads, and Employees.
*   **Lifecycle Management:** Track assets from acquisition to retirement (Available, Allocated, Reserved, Under Maintenance, Lost, Retired, Disposed).
*   **Smart Allocation:** Conflict-aware allocation system prevents double-booking and simplifies asset transfers.
*   **Resource Booking:** Overlap-validated scheduling for shared spaces and equipment.
*   **Maintenance & Audits:** Approval-based maintenance workflows and structured audit cycles with discrepancy reporting.
*   **Dashboard & Notifications:** Real-time operational insights via KPIs and automated activity alerts.

## 🏗️ Technical Architecture
*   **Stack:** MERN (MongoDB, Express, React, Node.js)
*   **Containerization:** Docker-based environment for seamless setup and scalability.
*   **Design:** Professional UI/UX using component libraries for an enterprise-ready look.

## 🛠️ Roles & Responsibilities
*   **Admin:** Organization master data (Departments, Categories, Employee Directory, Roles).
*   **Asset Manager:** Asset registration, approval workflows (transfers, maintenance), and return handling.
*   **Department Head:** Departmental oversight, resource booking, and internal approval.
*   **Employee:** Self-service resource booking, asset requests, and maintenance reporting.

## 📈 Workflow
1.  **Setup:** Admin configures the organizational structure.
2.  **Registration:** Asset Manager registers assets to the system.
3.  **Operations:** Assets are allocated, transferred, or returned based on business needs.
4.  **Maintenance:** Requests are raised and approved through a structured lifecycle.
5.  **Audit:** Periodic cycles verify asset presence and generate discrepancy reports.
6.  **Monitoring:** Dashboards and activity logs track all changes in real-time.
